## 快速开始

### 步骤1: 启动Gazebo仿真

在终端1中运行：
```bash
ros2 launch dog2_motion_control spider_gazebo_complete.launch.py
```

等待仿真完全启动（约10秒），直到看到机器人模型出现。

### 步骤2: 运行集成测试

在终端2中运行：
```bash
cd src/dog2_motion_control/test
./run_integration_tests.sh
```

或者手动运行单个测试：
```bash
# Gazebo仿真测试
python3 test_gazebo_simulation.py

# 导轨锁定验证
python3 test_rail_locking.py
```

## 测试参数


可以在测试文件中修改此阈值。

### 测试持续时间
- 静止测试: 5秒
- 运动测试: 10秒
- 紧急停止测试: 5秒
- 监控测试: 3秒

### 采样频率
- 目标频率: 50 Hz
- 实际频率: 取决于系统性能

## 故障排除

### 问题: 未检测到Gazebo仿真

**检查**:
```bash
# 检查话题
ros2 topic list | grep joint_states

# 检查控制器
ros2 control list_controllers
```

**解决**:
1. 确认Gazebo正在运行
2. 等待仿真完全启动
3. 检查控制器是否加载成功

### 问题: 导轨滑动超出阈值

**可能原因**:
1. 控制器增益不足
2. 仿真物理参数不正确
3. 负载过大

**解决**:
1. 调整PID参数（在ros2_controllers.yaml中）
2. 检查仿真时间步长
3. 减小运动速度

### 问题: 测试超时

**解决**:
1. 增加超时时间
2. 检查网络连接
3. 重启仿真
