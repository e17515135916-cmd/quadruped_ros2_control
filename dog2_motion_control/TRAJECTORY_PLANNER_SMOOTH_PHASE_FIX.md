# 轨迹规划器平滑相位修复

## 日期
2026-03-01

## 问题分析

### 原始实现的严重缺陷

原始代码使用 `scipy.interpolate.CubicSpline(..., bc_type='natural')` 生成轨迹。

**数学事实：**
- `bc_type='natural'` 只保证**二阶导数为零**（加速度）
- **不保证一阶导数为零**（速度）

**物理灾难：**
当拟合线性分布的点时（如支撑相的3个点），生成的样条是直线 `y = kx`：
- 起步时刻 `t=0`：足端以**非零初速度**瞬间启动
- 停止时刻 `t=duration`：足端以**非零速度**瞬间停止

在 Gazebo 物理引擎中：
- 速度阶跃 = 无穷大加速度
- 无穷大加速度 = 无穷大冲击力（Jerk）
- 结果：机器人腿部在起步和落地瞬间**疯狂抽搐甚至崩溃**

### 测试验证

原始实现的速度测试：
```
摆动相起点速度: 0.199800 m/s  ❌ 应该为 0
摆动相终点速度: 0.199800 m/s  ❌ 应该为 0
```

## 解决方案：无状态解析平滑相位

### 核心思想

使用**五次多项式**作为平滑相位函数，将线性时间 `s ∈ [0, 1]` 映射到平滑相位 `φ(s) ∈ [0, 1]`。

### 数学定义

五次多项式：
```
φ(s) = 6s⁵ - 15s⁴ + 10s³
```

**数学性质（严格保证）：**
1. `φ(0) = 0, φ(1) = 1` （边界条件）
2. `φ'(0) = 0, φ'(1) = 0` （**速度为零**）
3. `φ''(0) = 0, φ''(1) = 0` （**加速度为零**）

### 实现代码

```python
@staticmethod
def smooth_phase(s: float) -> float:
    """平滑相位函数（五次多项式）
    
    将线性相位 s ∈ [0, 1] 映射到平滑相位 φ(s) ∈ [0, 1]
    
    数学性质：
    - φ(0) = 0, φ(1) = 1
    - φ'(0) = 0, φ'(1) = 0  （速度为零）
    - φ''(0) = 0, φ''(1) = 0 （加速度为零）
    
    五次多项式：φ(s) = 6s^5 - 15s^4 + 10s^3
    """
    s = np.clip(s, 0.0, 1.0)
    return 6.0 * s**5 - 15.0 * s**4 + 10.0 * s**3
```

### 轨迹生成

**支撑相轨迹：**
```python
def trajectory(t: float) -> np.ndarray:
    s = np.clip(t / duration, 0.0, 1.0)
    phi = self.smooth_phase(s)  # 平滑相位
    
    # 所有维度使用平滑插值
    x = start_pos[0] + phi * (end_pos[0] - start_pos[0])
    y = start_pos[1] + phi * (end_pos[1] - start_pos[1])
    z = start_pos[2] + phi * (end_pos[2] - start_pos[2])
    
    return np.array([x, y, z])
```

**摆动相轨迹：**
```python
def trajectory(t: float) -> np.ndarray:
    s = np.clip(t / duration, 0.0, 1.0)
    phi = self.smooth_phase(s)  # 平滑相位
    
    # 水平方向：平滑插值
    x = start_pos[0] + phi * (end_pos[0] - start_pos[0])
    y = start_pos[1] + phi * (end_pos[1] - start_pos[1])
    
    # 垂直方向：平滑插值 + 平滑抛物线抬起
    z_base = start_pos[2] + phi * (end_pos[2] - start_pos[2])
    z_lift = 4.0 * height * phi * (1.0 - phi)  # 使用 phi 而非 s
    z = z_base + z_lift
    
    return np.array([x, y, z])
```
