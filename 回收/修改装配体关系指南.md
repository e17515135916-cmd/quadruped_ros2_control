# 修改Dog2机器人装配体关系指南

## 问题理解

你需要修改的是**装配关系**（部件之间的相对位置和旋转），而不是mesh的几何形状。

## ✅ 最佳方案：直接编辑URDF + RViz实时查看

### 为什么这是最好的方案？

1. **装配关系存储在URDF中**，不在mesh文件里
2. **RViz可以实时显示**修改后的效果
3. **不需要CAD软件**，直接编辑文本文件
4. **精确控制**每个关节的位置和旋转

## 🔧 工作流程

### 步骤1：在RViz中查看当前装配体

```bash
cd ~/aperfect/carbot_ws
source install/setup.bash
ros2 launch dog2_description view_dog2.launch.py
```

在RViz中观察哪些部件的位置不对。

### 步骤2：编辑URDF文件

URDF文件位置：`src/dog2_description/urdf/dog2.urdf.xacro`

**关键参数：**

```xml
<!-- 每个joint都有一个origin标签，控制装配关系 -->
<joint name="j11" type="revolute">
  <origin rpy="0 0 1.5708" xyz="-0.016 0.0199 0.055"/>
  <!--     ↑ 旋转角度        ↑ 位置偏移           -->
  <parent link="l1"/>
  <child link="l11"/>
  <axis xyz="0 0 -1"/>
</joint>
```

**参数说明：**
- `xyz` - 位置偏移（米）：X Y Z
- `rpy` - 旋转角度（弧度）：Roll Pitch Yaw
- `axis` - 关节旋转轴

### 步骤3：重新编译并查看

```bash
colcon build --packages-select dog2_description
source install/setup.bash
ros2 launch dog2_description view_dog2.launch.py
```

### 步骤4：迭代调整

重复步骤2-3，直到装配关系正确。

## 📐 常见装配调整

### 调整部件位置

修改`xyz`参数：

```xml
<!-- 原来 -->
<origin rpy="0 0 1.5708" xyz="-0.016 0.0199 0.055"/>

<!-- 向X轴正方向移动0.01米 -->
<origin rpy="0 0 1.5708" xyz="-0.006 0.0199 0.055"/>
```

### 调整部件旋转

修改`rpy`参数（注意：使用弧度）：

```xml
<!-- 原来 -->
<origin rpy="0 0 1.5708" xyz="-0.016 0.0199 0.055"/>

<!-- 绕Z轴再旋转90度（1.5708弧度） -->
<origin rpy="0 0 3.1416" xyz="-0.016 0.0199 0.055"/>
```

**常用角度转换：**
- 90° = 1.5708 rad
- 180° = 3.1416 rad
- 270° = 4.7124 rad
- 360° = 6.2832 rad

### 调整关节轴方向

修改`axis`参数：

```xml
<!-- X轴旋转 -->
<axis xyz="1 0 0"/>

<!-- Y轴旋转 -->
<axis xyz="0 1 0"/>

<!-- Z轴旋转 -->
<axis xyz="0 0 1"/>

<!-- 反向Z轴旋转 -->
<axis xyz="0 0 -1"/>
```

## 🎯 你的具体案例

### 已完成的髋关节改造

你已经完成了ZYY到XYY的改造：

```xml
<!-- 髋关节 (j11, j21, j31, j41) -->
<joint name="j${leg_num}1" type="revolute">
  <origin rpy="${hip_joint_rpy}" xyz="${hip_joint_xyz}"/>
  <parent link="l${leg_num}"/>
  <child link="l${leg_num}1"/>
  <axis xyz="0 0 -1"/>  <!-- 已改为X轴旋转 -->
  <limit effort="${hip_effort}" lower="${hip_lower_limit}" 
         upper="${hip_upper_limit}" velocity="${hip_velocity}"/>
</joint>
```

### 如果需要进一步调整

**例如：调整髋关节连接点位置**

1. 在RViz中观察当前位置
2. 编辑`dog2.urdf.xacro`中的`hip_joint_xyz`参数
3. 重新编译并查看效果

```xml
<!-- 在macro定义中 -->
<xacro:macro name="leg" params="... hip_joint_xyz:='-0.016 0.0199 0.055' ...">
  
  <!-- 使用时可以覆盖 -->
  <xacro:leg leg_num="1" 
             ...
             hip_joint_xyz="-0.020 0.0199 0.055"/>  <!-- 修改X坐标 -->
</xacro:macro>
```

## 💡 实用技巧

### 1. 使用变量简化修改

在URDF顶部定义变量：

```xml
<xacro:property name="hip_offset_x" value="-0.016"/>
<xacro:property name="hip_offset_y" value="0.0199"/>
<xacro:property name="hip_offset_z" value="0.055"/>

<!-- 使用变量 -->
<origin rpy="0 0 1.5708" xyz="${hip_offset_x} ${hip_offset_y} ${hip_offset_z}"/>
```

### 2. 小步调整

每次只修改一个参数，修改量不要太大（例如±0.005米）。

### 3. 记录修改

在URDF文件顶部的注释中记录每次修改：

```xml
<!--
  修改历史：
  2026-02-01: 髋关节从ZYY改为XYY
  2026-02-01: 调整j41位置，X从0.0116改为-0.016
-->
```

### 4. 使用RViz的TF显示

在RViz中启用TF显示，可以看到所有坐标系：

1. Add → TF
2. 查看每个link的坐标系
3. 确认装配关系是否正确

## 🔄 完整工作流程示例

假设你发现l11（蓝色髋关节）位置不对：

```bash
# 1. 在RViz中查看
ros2 launch dog2_description view_dog2.launch.py

# 2. 编辑URDF（在另一个终端）
nano src/dog2_description/urdf/dog2.urdf.xacro

# 找到hip_joint_xyz参数，修改数值
# 例如：xyz="-0.016 0.0199 0.055" 改为 xyz="-0.020 0.0199 0.055"

# 3. 保存并重新编译
colcon build --packages-select dog2_description
source install/setup.bash

# 4. 刷新RViz（或重启）
# 在RViz中：File → Reload Config
# 或者关闭RViz重新启动

# 5. 检查效果，如果不满意，重复步骤2-4
```

## ⚠️ 注意事项

1. **备份URDF文件**
   ```bash
   cp src/dog2_description/urdf/dog2.urdf.xacro \
      src/dog2_description/urdf/dog2.urdf.xacro.backup_$(date +%Y%m%d_%H%M%S)
   ```

2. **角度使用弧度**
   - URDF中的角度都是弧度，不是度数
   - 使用计算器：度数 × π / 180 = 弧度

3. **坐标系方向**
   - X轴：红色，通常指向前方
   - Y轴：绿色，通常指向左侧
   - Z轴：蓝色，通常指向上方

4. **编译后必须source**
   ```bash
   source install/setup.bash
   ```
