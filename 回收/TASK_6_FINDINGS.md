# 任务6 RViz测试发现

## 日期：2026-02-07

## 发现的问题

在RViz中测试髋关节支架时，发现了一个关键问题：

### 问题：髋关节仍然绕Z轴旋转

虽然关节轴（axis）已经正确设置为X轴：
```xml
<axis xyz="1 0 0"/>
```

但是关节的origin RPY旋转仍然包含90度的Z轴旋转：

**前腿（腿1和腿2）：**
- 使用默认值：`hip_joint_rpy="0 0 1.5708"`
- 这是90度的Z轴旋转

**后腿（腿3和腿4）：**
- 明确设置：`hip_joint_rpy="3.1416 0 1.5708"`
- 这是180度X轴旋转 + 90度Z轴旋转

### 影响

这个90度的Z轴RPY旋转会导致：
1. 关节的局部坐标系被旋转了90度
2. 虽然axis定义为X轴，但由于坐标系旋转，实际旋转轴变成了原来的Z轴
3. 结果：髋关节仍然是"蜘蛛式"的左右旋转，而不是"狗式"的前后旋转

### 解决方案

需要完成**任务5：移除RPY旋转补偿**：

1. **前腿（腿1和腿2）：**
   - 当前：`hip_joint_rpy="0 0 1.5708"`
   - 目标：`hip_joint_rpy="0 0 0"`

2. **后腿（腿3和腿4）：**
   - 当前：`hip_joint_rpy="3.1416 0 1.5708"`
   - 目标：`hip_joint_rpy="3.1416 0 0"`

### 任务顺序建议

应该先完成任务5（移除RPY旋转），然后再继续任务6的验证。
