# HAA 关节修复说明

## 问题

您正确地观察到：**HAA、HFE、KFE 三个关节都在同一个平面旋转**。

这确实**不符合** CHAMP 框架标准！

## 原因

1. **滑动副（Prismatic Joint）的旋转**
   - 滑动副有一个 90° 的旋转 (`rpy="1.5708 0 0"`)
   - 这导致整个腿部坐标系被旋转
   - HAA 关节的 z 轴也被旋转了

2. **HAA 关节的默认旋转**
   - 之前的默认值是 `'0 0 1.5708'` (0° 0° 90°)
   - 这进一步扭曲了 HAA 的旋转轴
   - 结果：HAA、HFE、KFE 都在同一平面旋转

## 修复方案

我已经进行了以下修复：

### 1. 修改默认值

将 HAA 关节的默认旋转从 `'0 0 1.5708'` 改为 `'0 0 0'`

### 2. 添加补偿旋转

为前两条腿（Leg 1 和 Leg 2）添加了补偿旋转 `-1.5708 0 0` (即 -90° 0° 0°)

**原理**：
- 滑动副旋转了 +90°
- HAA 关节反向旋转 -90° 来补偿
- 这样 HAA 的 z 轴就能垂直向上

### 3. 简化 HFE 旋转

将 HFE 关节的旋转从 `'1.5708 1.5708 0'` 改为 `'0 1.5708 0'`

## 修复结果

现在编译后的 URDF 显示：

- **前左腿 (lf_haa_joint)**: `RPY = -1.5708 0 0` ✓
- **前右腿 (rf_haa_joint)**: `RPY = -1.5708 0 0` ✓

   ```bash

   ```

3. **观察前左腿的运动**：
   - ✓ 腿部应该**左右摆动**（外展/内收）
   - ✓ 运动应该**垂直于身体平面**
   - ✓ 整条腿一起摆动

## 预期效果

修复后，三个关节应该有不同的功能：

1. **HAA (Hip Abduction/Adduction)**
   - 功能：腿部外展/内收
   - 效果：腿部左右摆动
   - 方向：垂直于腿部平面

2. **HFE (Hip Flexion/Extension)**
   - 功能：髋关节前后摆动
   - 效果：大腿前后摆动
   - 方向：在腿部平面内

3. **KFE (Knee Flexion/Extension)**
   - 功能：膝关节屈伸
   - 效果：小腿折叠
   - 方向：在腿部平面内，与 HFE 平行

## 判断标准

- ✅ **正确**: HAA 控制外展/内收（腿部左右摆动）
- ❌ **错误**: HAA、HFE、KFE 都在同一平面旋转
