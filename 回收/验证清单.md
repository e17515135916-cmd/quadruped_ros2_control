## 🔍 故障排查

### 如果 quadruped_controller 仍未启动

#### 检查 1：查看 Gazebo 终端错误
在 Gazebo 终端查找包含 "quadruped" 或 "champ" 的错误信息

#### 检查 2：手动测试节点
```bash
ros2 node list | grep quadruped
```
如果没有输出，说明节点未启动

#### 检查 3：查看节点日志
```bash
ros2 node info /quadruped_controller
```

#### 检查 4：验证配置文件
```bash
ls -la install/dog2_champ_config/share/dog2_champ_config/config/
```
应该看到 gait/, joints/, links/ 三个目录

---

### 如果机器人不动

#### 检查 1：验证 /cmd_vel 消息
```bash
ros2 topic echo /cmd_vel
```
按键盘时应该看到消息输出

#### 检查 2：验证关节轨迹命令
```bash
ros2 topic echo /joint_trajectory_controller/joint_trajectory
```
按键盘时应该看到轨迹命令

#### 检查 3：手动发送速度命令
```bash
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.3, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}" --once
```
机器人应该移动

#### 检查 4：查看关节状态
```bash
ros2 topic echo /joint_states
```
应该看到 12 个腿部关节的状态更新
