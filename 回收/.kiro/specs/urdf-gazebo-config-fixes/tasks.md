# 实现计划: URDF Gazebo 配置修复

## 概述

本实现计划将修复 dog2_visual.urdf 和 dog2_gazebo.urdf 文件中的严重 Gazebo 配置错误。实现采用 Python 语言，使用 xml.etree.ElementTree 库进行 XML 处理。

## 任务

- [ ] 1. 创建项目结构和核心类
  - 创建 `scripts/urdf_fixer/` 目录
  - 实现 `URDFFixer` 基础类，包含文件读取和保存功能
  - 实现 `InertiaExtractor` 类，用于从参考 URDF 提取惯性参数
  - 实现 `URDFValidator` 类，用于验证修复结果
  - 添加必要的导入和类型注解
  - _需求: 1.1, 2.1, 3.1, 4.1, 5.1, 6.1_

- [ ] 2. 实现 ROS 1 插件移除功能
  - [ ] 2.1 在 URDFFixer 中实现 `remove_ros1_plugin()` 方法
    - 使用 XPath 查找所有 `<plugin name="gazebo_ros_control">` 元素
    - 移除找到的插件元素及其父 `<gazebo>` 标签（如果该标签只包含此插件）
    - 记录移除的插件数量
    - _需求: 1.1, 1.4_

  - [ ]* 2.2 编写属性测试验证 ROS 1 插件完全移除
    - **属性 1: ROS 1 插件完全移除**
    - **验证: 需求 1.1, 1.2, 1.3, 1.4**

- [ ] 3. 实现 ROS 2 控制器配置路径修复
  - [ ] 3.1 在 URDFFixer 中实现 `fix_ros2_control_config()` 方法
    - 查找 `<plugin name="gazebo_ros2_control">` 元素
    - 移除其中的 `<parameters>` 子元素
    - 保留插件的其他配置
    - _需求: 2.1, 2.3, 2.4_

  - [ ]* 3.2 编写属性测试验证配置路径不包含占位符
    - **属性 2: ROS 2 控制器配置路径不包含占位符**
    - **验证: 需求 2.1, 2.3, 2.4**

- [ ] 4. 实现惯性参数恢复功能
  - [ ] 4.1 在 InertiaExtractor 中实现参数提取方法
    - 实现 `extract_link_inertia(link_name)` 方法
    - 实现 `get_all_inertias()` 方法
    - 解析 `<inertial>` 元素，提取 mass、origin、inertia 值
    - 返回结构化的惯性参数字典
    - _需求: 3.1, 3.5_

  - [ ] 4.2 在 URDFFixer 中实现 `restore_inertia_parameters()` 方法
    - 接受参考 URDF 路径作为参数
    - 使用 InertiaExtractor 从参考文件提取惯性参数
    - 替换当前 URDF 中所有 link 的 `<inertial>` 元素
    - 更新文件开头的注释，移除"质量值已缩放"警告
    - _需求: 3.2, 3.4, 3.5_

  - [ ]* 4.3 编写属性测试验证惯性参数一致性
    - **属性 3: 惯性参数恢复一致性**
    - **验证: 需求 3.1, 3.2, 3.5**

- [ ] 5. 检查点 - 确保核心修复功能正常
  - 确保所有测试通过，如有问题请询问用户

- [ ] 6. 实现接触传感器名称修复
  - [ ] 6.1 在 URDFFixer 中实现 `fix_contact_sensor_names()` 方法
    - 查找所有 `<sensor type="contact">` 元素
    - 将碰撞体名称从复杂格式简化为 `{link_name}_collision`
    - 例如: `l1111_fixed_joint_lump__l1111_collision_collision` → `l1111_collision`
    - 处理所有四条腿的足端传感器 (l1111, l2111, l3111, l4111)
    - _需求: 4.2, 4.4_

  - [ ]* 6.2 编写属性测试验证传感器名称格式
    - **属性 4: 接触传感器碰撞体名称简化**
    - **验证: 需求 4.2, 4.4**

- [ ] 7. 实现接触刚度调整功能
  - [ ] 7.1 在 URDFFixer 中实现 `adjust_contact_stiffness()` 方法
    - 查找所有足端 `<gazebo reference="l*111">` 元素中的 `<kp>` 标签
    - 将 kp 值从 1000000.0 降低到 10000.0（可配置）
    - 保持 kd、mu1、mu2 等其他参数不变
    - _需求: 5.2, 5.4_

  - [ ]* 7.2 编写属性测试验证刚度在安全范围
    - **属性 5: 接触刚度在安全范围内**
    - **验证: 需求 5.2, 5.4**

    - 实现 `check_ros2_control_config()` - 验证配置路径
    - 实现 `check_inertia_parameters()` - 验证惯性参数
    - 实现 `check_contact_sensors()` - 验证传感器配置
    - 实现 `check_contact_stiffness()` - 验证刚度范围
    - 实现 `validate_all()` - 运行所有检查并汇总结果
    - _需求: 6.1, 6.2, 6.3, 6.4_

  - [ ]* 8.2 编写单元测试验证各个验证方法
