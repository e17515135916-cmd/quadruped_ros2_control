# J21位置修复总结

## 修复时间
2026-02-02 15:30:32

## 问题描述
第二条腿（右前腿）的j21关节位置不正确，在RViz中显示位置偏离。

## 根本原因
Leg 2没有指定`hip_xyz`参数，使用了默认值`-0.016 0.0199 0.055`，这个值与Leg 1相同。但是由于左右腿的对称性，右腿的hip_xyz的X分量应该与左腿相反。

```xml
           p_low="-0.111" p_up="0.111"
           thigh_xyz="${leg12_thigh_xyz}" shin_xyz="${leg12_shin_xyz}"/>
           <!-- hip_xyz使用默认值: -0.016 0.0199 0.055 -->

<!-- Leg 2: 右前腿 -->
<xacro:leg leg_num="2" xyz="1.3491 -0.80953 0.2649" rpy="1.5708 0 0" 
           p_low="-0.111" p_up="0.111"
           thigh_xyz="${leg12_thigh_xyz}" shin_xyz="${leg12_shin_xyz}"/>
           <!-- hip_xyz使用默认值: -0.016 0.0199 0.055 ← 问题所在 -->
```

### 修复后
```xml
<!-- Leg 1: 左前腿 -->
<xacro:leg leg_num="1" xyz="1.1026 -0.80953 0.2649" rpy="1.5708 0 0" 
           p_low="-0.111" p_up="0.111"
           thigh_xyz="${leg12_thigh_xyz}" shin_xyz="${leg12_shin_xyz}"/>
           <!-- hip_xyz: -0.016 0.0199 0.055 -->

<!-- Leg 2: 右前腿 -->
<xacro:leg leg_num="2" xyz="1.3491 -0.80953 0.2649" rpy="1.5708 0 0" 
           p_low="-0.111" p_up="0.111"
           thigh_xyz="${leg12_thigh_xyz}" shin_xyz="${leg12_shin_xyz}"
           hip_xyz="0.016 0.0199 0.055"/>
           <!-- ✓ 添加了自定义hip_xyz，X分量取反 -->
```

## 修复逻辑

通过分析四条腿的配置发现：

1. **Leg 1 (左前)**：`hip_xyz = -0.016 0.0199 0.055` (默认值)
2. **Leg 2 (右前)**：`hip_xyz = -0.016 0.0199 0.055` (默认值) ← 错误
3. **Leg 3 (左后)**：`hip_xyz = -0.016 0.0199 0.055` (默认值)
4. **Leg 4 (右后)**：`hip_xyz = 0.0116 0.0199 0.055` (自定义，X取反)

**关键发现**：
- Leg 4（右后腿）有自定义的`hip_xyz`，其X分量从`-0.016`变为`0.0116`
- 这表明右侧腿的hip_xyz的X分量应该与左侧相反
- 因此Leg 2（右前腿）也应该有相反的X分量

**修复方案**：
- 为Leg 2添加`hip_xyz="0.016 0.0199 0.055"`
- X分量从`-0.016`变为`0.016`（取反）
- Y和Z分量保持不变

## 技术细节

### Hip Joint位置参数说明

`hip_xyz`参数定义了髋关节（j${leg_num}1）相对于prismatic link（l${leg_num}）的位置：

- **X分量**：前后方向
  - 负值：向后
  - 正值：向前
  - 左右腿应该相反（对称）

- **Y分量**：左右方向
  - 负值：向左
  - 正值：向右
  - 左右腿应该相同

- **Z分量**：上下方向
  - 负值：向下
  - 正值：向上
  - 左右腿应该相同

### 对称性原则

对于四足机器人：
- **前腿对称**：Leg 1（左前）和Leg 2（右前）应该关于机身中线对称
- **后腿对称**：Leg 3（左后）和Leg 4（右后）应该关于机身中线对称
- **左右对称**：X分量取反，Y和Z分量相同
