## 📖 阶段1：机身前探 (Body Forward Shift)

### 时间：t = 10s ~ 12s

### 目标
```
将机身向前推进，使前腿滑动副的前端穿过窗框，
但腿的根部（连接点）仍停留在窗框外侧。
```

### 初始状态（t = 10s）
```
位置：
- 质心：x = 1.8m, z = 0.3m
- 前腿滑动副前端：x ≈ 1.9m
- 窗框：x = 2.0m

构型：
- 四腿：全部肘式
- 滑动副：d1=d2=d3=d4=0
- 四腿：全部接触地面（静态支撑）
```


### 动作序列

#### 步骤1.1：四腿锁定 (t = 10.0s)
```
动作：
- 四条腿的足端位置固定不动
- 增大足端与地面的摩擦力
- 准备承受机身推进产生的反作用力

关节控制：
- 所有旋转关节：位置保持模式
- 所有滑动副：准备伸展

力学状态：
- 每条腿的接触力：Fz ≈ 20N（支撑重力）
- 摩擦系数：μ ≥ 0.5
- 支撑多边形：稳定
```

#### 步骤1.2：前腿滑动副伸展 (t = 10.0s ~ 11.0s)
```
控制指令：
- j1（腿1滑动副）：d1 = 0 → -0.111m（向后滑动，相对机身）
- j4（腿4滑动副）：d4 = 0 → -0.111m（向后滑动，相对机身）
- 速度：v_slide = 0.111 m/s（线性加速和减速）

物理效果：
- 腿1和腿4的根部相对机身向后移动
- 由于足端固定，机身被向前推
- 机身前移距离：Δx ≈ 0.111m

时间分解：
  t = 10.0s ~ 10.1s：加速阶段
    - d1, d4: 0 → -0.011m
    - 机身位移：Δx = 0.011m
  
  t = 10.1s ~ 10.9s：匀速阶段
    - d1, d4: -0.011m → -0.100m
    - 机身位移：Δx = 0.089m
  
  t = 10.9s ~ 11.0s：减速阶段
    - d1, d4: -0.100m → -0.111m
    - 机身位移：Δx = 0.011m

力学分析：
- 滑动副推力：F_slide ≈ 30N（每条腿）
- 足端摩擦力：F_friction = F_slide（平衡）
- 质心加速度：ax ≈ 0.5 m/s²（加速阶段）
```

#### 步骤1.3：后腿滑动副协调伸展 (t = 10.0s ~ 11.0s)
```
控制指令：
- j2（腿2滑动副）：d2 = 0 → 0.111m（向前滑动，相对机身）
- j3（腿3滑动副）：d3 = 0 → 0.111m（向前滑动，相对机身）
- 速度：v_slide = 0.111 m/s

物理效果：
- 腿2和腿3的根部相对机身向前移动
- 由于足端固定，这也推动机身向前
- 与前腿协同作用

协调原理：
- 前腿向后滑 + 后腿向前滑 = 双向推进
- 增大推进效率
- 保持机身姿态稳定（pitch角不变）
```

#### 步骤1.4：质心前移完成 (t = 11.0s)
```
最终位置：
- 质心：x = 1.8m + 0.111m = 1.911m
- 前腿滑动副前端：x ≈ 2.011m（已穿过窗框！）
- 前腿根部：x ≈ 1.9m（仍在窗框外）
- 窗框：x = 2.0m

验证：
✓ 前腿滑动副前端 > 窗框位置（2.011m > 2.0m）
✓ 前腿根部 < 窗框位置（1.9m < 2.0m）
✓ 为前腿穿越创造了条件

构型状态：
- 四腿：全部肘式
- 滑动副：d1=-0.111m, d2=0.111m, d3=0.111m, d4=-0.111m
- 四腿：全部接触地面
```


#### 步骤1.5：MPC控制挑战
```
控制难点：
1. 质心极度前移
   - 质心从机身中心向前偏移
   - 支撑多边形不变
   - 容易产生pitch前倾

2. 静态稳定性
   - 质心投影必须在支撑多边形内
   - 需要精确控制pitch力矩
   - 防止机身低头或抬头

3. 滑动副协调
   - 前后腿滑动副必须同步
   - 速度曲线必须匹配
   - 防止机身扭转

MPC求解：
- 状态：x = [x, y, z, roll, pitch, yaw, vx, vy, vz, wx, wy, wz]
- 控制：u = [f1x, f1y, f1z, f2x, f2y, f2z, f3x, f3y, f3z, f4x, f4y, f4z]
- 约束：
  * 滑动副位置约束
  * 滑动副速度约束
  * 质心稳定性约束
  * pitch角约束：|pitch| < 5°
- 目标：
  * 质心平滑前移
  * 姿态保持水平
  * 最小化控制能量
```
