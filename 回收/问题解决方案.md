# 问题解决方案

## 遇到的问题

### 问题1：Blender脚本参数错误
```
工作空间: --
❌ 工作空间不存在: --
```

**原因**：脚本传递参数给Blender时格式错误

**已修复**：更新了脚本参数传递方式

### 问题2：编译失败 - 重复的包
```
ERROR:colcon:colcon build: Duplicate package names not supported:
- dog2_description:
  - backups/back2.1/dog2_description
  - src/dog2_description
```

**原因**：`backups/back2.1/` 文件夹中有另一个 `dog2_description` 包，colcon不允许重复的包名

**解决方案**：在 `backups` 文件夹中创建 `COLCON_IGNORE` 文件，让colcon忽略这个文件夹

## ⚠️ 关于COLCON_IGNORE

`COLCON_IGNORE` 是一个特殊文件，告诉colcon忽略某个文件夹。

**创建方法：**
```bash
touch backups/COLCON_IGNORE
```

**效果：**
- colcon会跳过 `backups/` 文件夹
- 不会扫描其中的包
- 解决重复包名问题

**不影响：**
- 文件夹中的文件仍然存在
- 只是colcon不会处理它们
