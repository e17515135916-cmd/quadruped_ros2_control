# RViz方向问题分析

## 问题描述

移除髋关节的90度Z轴RPY旋转后，机器人在RViz中的姿态变得奇怪，角度不对。

## 根本原因

原来的设计中，各个关节的RPY旋转是相互配合的：

### 原来的配置（蜘蛛式）：
```
髋关节 (j${leg_num}1):
  - hip_rpy = "0 0 1.5708"  (90度Z轴旋转)
  - axis = "1 0 0"  (但因为RPY旋转，实际是Z轴)

膝关节 (j${leg_num}11):
  - rpy = "1.5708 1.5708 0"  (90度X + 90度Y)
  - axis = "-1 0 0"
```

这些旋转是相互配合的，形成了"蜘蛛式"的腿部方向。

### 现在的配置（不完整的狗式）：
```
髋关节 (j${leg_num}1):
  - hip_rpy = "0 0 0"  (移除了Z轴旋转)
  - axis = "1 0 0"  (真正的X轴)

膝关节 (j${leg_num}11):
  - rpy = "1.5708 1.5708 0"  (还是原来的！)
  - axis = "-1 0 0"
```

**问题**：膝关节的RPY还是为原来的蜘蛛式设计的，导致大腿的方向不对。

## 解决方案

需要调整膝关节的RPY来配合新的髋关节X轴旋转。

### 方案1：调整膝关节RPY为 "0 1.5708 0"

这会让大腿绕Y轴旋转90度，使其指向正确的方向。

```xml
<joint name="j${leg_num}11" type="revolute">
  <origin rpy="0 1.5708 0" xyz="${knee_xyz}"/>
  <parent link="l${leg_num}1"/>
  <child link="l${leg_num}11"/>
  <axis xyz="-1 0 0"/>
</joint>
```

### 方案2：调整膝关节RPY为 "1.5708 0 0"

这会让大腿绕X轴旋转90度。

```xml
<joint name="j${leg_num}11" type="revolute">
  <origin rpy="1.5708 0 0" xyz="${knee_xyz}"/>
  <parent link="l${leg_num}1"/>
  <child link="l${leg_num}11"/>
  <axis xyz="-1 0 0"/>
</joint>
```

### 方案3：完全移除膝关节RPY

```xml
<joint name="j${leg_num}11" type="revolute">
  <origin rpy="0 0 0" xyz="${knee_xyz}"/>
  <parent link="l${leg_num}1"/>
  <child link="l${leg_num}11"/>
  <axis xyz="-1 0 0"/>
</joint>
```

## 建议的测试步骤

1. 先尝试方案2（`rpy="1.5708 0 0"`）
2. 如果不对，尝试方案1（`rpy="0 1.5708 0"`）
3. 如果还不对，尝试方案3（`rpy="0 0 0"`）
4. 每次修改后重新构建并在RViz中测试

## 坐标系分析

原来的蜘蛛式设计：
- 髋关节：Z轴旋转（左右摆动）
- 膝关节：需要复杂的RPY来让大腿指向侧面

新的狗式设计：
- 髋关节：X轴旋转（前后摆动）
- 膝关节：需要简单的RPY来让大腿指向下方
