# RPY旋转问题分析

## 日期：2026-02-07

## 问题总结

尝试将髋关节从Z轴旋转改为X轴旋转时，遇到了复杂的坐标系变换问题。

## 腿部配置（正确）

- **腿1（左前）**：j1（棱柱）, j11（髋）, j111（膝）
- **腿2（左后）**：j2（棱柱）, j21（髋）, j211（膝）
- **腿3（右后）**：j3（棱柱）, j31（髋）, j311（膝）
- **腿4（右前）**：j4（棱柱）, j41（髋）, j411（膝）

**前腿**：j11, j41
**后腿**：j21, j31

## 原始设计（蜘蛛式）

```xml
<!-- 髋关节 -->
<joint name="j${leg_num}1">
  <origin rpy="0 0 1.5708" xyz="..."/>  <!-- 90度Z轴旋转 -->
  <axis xyz="1 0 0"/>  <!-- 但因为RPY，实际是Z轴 -->
</joint>

<!-- 膝关节 -->
<joint name="j${leg_num}11">
  <origin rpy="1.5708 1.5708 0" xyz="..."/>  <!-- 90度X + 90度Y -->
  <axis xyz="-1 0 0"/>
</joint>

<!-- 后腿额外的180度X轴旋转 -->
腿3和腿4: hip_rpy="3.1416 0 1.5708"
```

## 尝试的修改

### 修改1：移除髋关节Z轴旋转
- 前腿：`hip_rpy="0 0 0"`
- 后腿：`hip_rpy="3.1416 0 0"`

**结果**：腿向侧面伸出（蜘蛛式）

### 修改2：调整膝关节RPY
- 所有腿：`knee_rpy="0 1.5708 0"`

**结果**：腿向下了，但前后腿方向相反

### 修改3：为后腿使用不同的knee_rpy
- 前腿：`knee_rpy="0 1.5708 0"`
- 后腿：`knee_rpy="0 -1.5708 0"` 或 `"0 1.5708 3.1416"`

**结果**：还是不对，而且Z轴位置不齐平

## 根本问题

1. **坐标系复杂性**：原始设计使用了多层RPY旋转来实现蜘蛛式配置
2. **前后腿不对称**：后腿有额外的180度X轴旋转
3. **mesh方向**：大腿和小腿的mesh是为蜘蛛式设计的

## 可能的解决方案

**优点**：
- 已知可以工作
- 不需要复杂的坐标变换
- 可以继续其他任务

**缺点**：

**说明**：

- 关节轴已经改为X轴
- 虽然腿的方向还不完美，但证明了概念
